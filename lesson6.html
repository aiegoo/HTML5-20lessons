Using hyperlinks

To create a link to another document , you must first know where this new document resides , whether it is on the local server or another one.An external web address uses what is known as an absolute URL which usually begins with https://.
http://bbc.co.uk (simple URL)
https://bbc.co.uk/weather/ (include folder structure within webpage resides which accesses the folder /weather)
http://www.bbc.co.uk (some server add prefix before the domain name, the most common of which is www)
http://news.bbc.co.uk (some servers may use prefixes such as news)

The Query String

Many websites support the use of a query string, which is a string of data placed after the URL to provide additional information to a web server.
for example: https://google.com/search?q=html5
URL passes the value html5 to the field name q on Google search page.Query String begin with a ? character, followed by a field if preceded by a & character, like this:
https://google.com/search?q=html5&h1=fr here field name h1 is set to the value fr.

Relative URLs

when documents reside on the current server, you can access then more easily and make them more portable by using relative instead of absolute URLs.
for example: if you need to link to the file agenda.htm in the current folder, you can simply use its name as URL : agenda.htm.
if the document is in the subfolder mettings, you can refer to it like this: meetings/agenda.htm.
if the file is in parent folder of the current one, you can use specail notation: ../agenda.htm.
finally , if you refer to root folder of current drive or web location use / symbol. -> /agenda.htm.
