1. New Form Attributes

The autocomplete Attribute:

  • it can applied to form tag or any of the color,date email,password, range,search,telephone,text or url types of the input tag. when autocomplete is on, any field with an id that has previously had an input entered will remember its value and offer it as a suggested value, saving you from having to enter it again

The autofocus Attribute

  • can be applied to input tag to give its fields automatic focus when a page loads. to achieve the same effect on older browsers you can add some javascript to body tag of your web page: onload='document.forms.myform.myfield.focus()'
Text

The form attribute

  • with the form attribute, it is not necessary for you to place input tag wihn the form to which they apply. As long as you give form an ID, you can specify that value as argumant for the form attribute.

2. Form Overrides

The formaction Attribute

  • It let's you change the action attribute to different destination.
  • for example the form will not post to program "prog.php" as specified in the form , instead post to "prog2.php"

The formenctype Attribute

  • It let's to change the encoding type of the form
  • it is similar to formaction attribute

The formmethod Attribute

  • It let's you to change the posting method(post or get) in a similar manner to the formaction method

The formnovalidate Attribute

  • It let's you change the novalidate attribute.

The formtarget Attribute

  • It let's you change the target attribute.

The height and width Attribute

  • The height and width attribute can be applied to image type of the input tag to change its height and width.

The listATtribute and datalist tag and option tags

  • datalist offer a selection of URLs from which to choose
  • The value supplied to list attribute should be the id name of datalist tag
Choose a web page:

The min and max Attribute

  • It is used to specify minimum and maximum value for input types that contain numbers and dates

The multiple Attribute

  • It allows you to accept multiple values for an input tag that uses any of the email,range, or file types.

The novalidate and formnovalidate Attributes

  • The novalidate attribute is applicableto form tag
  • The formnovalidate attribute is applicable to only submit and image types of the input tag.
Text

The pattern Attribute

  • It let's you to specify a regular expression with which an input field should be evaluated.It can be applied to input tag that uses any of the email,password,search,telephone,text or url types.

The placeholder Attribute

  • It let's you place a helpful hint in any blank input field with which you can help explian to users what they should enter.

The required Attribute

  • It is used to ensure that a field has been completed before a form is submitted.

The step Attribute

  • Itis used to specify a step value for input types that contain numbers or dates.

The date Input Type

  • It selects a date and returned value will be of the form YYYY-MM-DD.

the Month Input Type

  • It selects a month and returned value will be of the form YYYY-MM.

The time Input Type

  • It returns the time in the 24 hour form HH:MM.

The week Input Type

  • It returns the week in the form YYYY-WNN.

The datetime Input Type

  • It returns the date and time in UTC.

The datetime-local Input Type

  • It returns the user local date and time the returned value will be of the form YYYY-MM-DDTHH:MM.

The email Input Type

  • This type ensures that the browser knows an email address is excepted and if necessary can cater to it.

The number Input Type

  • It ensures that only numbers can be entered.

The range Input Type

  • It causes the range widget to be displayed that you can slide to select any value between min and max, with a specified start and step value.

The search Input Type

  • when you specify the search type, browsers are supposed to tailor the input box to provide features that might include search suggestions and an icon with which to empty the field, and possibly styling changes to alrt you to the type of input.

The tel Input Type

  • It informs the browser that a telephone number is to be expected.

The url Input Type

  • url type is also there to tell the browser about the type of data to be expected. It ensures that the .,/ and .com buttons are displayed.
